Linux 命令行快捷键
涉及在Linux命令行下进行快速移动光标、命令编辑、编辑后执行历史命令、Bang(!)命令、控制命令等，有效提高工作效率。

说明

  • Ctrl - k : 先按住 Ctrl 键，然后再按 k 键
  • Ctrl - kl : 先按住 Ctrl 键，然后再按 k 键，然后松开 k 键，再按 l 键
  • Alt - k : 先按住 Alt 键，然后再按 k 键
  • M - k : 先单击 Meta(Esc) 键，然后再按 k 键

移动光标

  • Ctrl - 左右键 : 在单词之间跳转
  • Ctrl - a : 移到行首
  • Ctrl - e : 移到行尾
  • Ctrl - b : 往回(左)移动一个字符
  • Ctrl - f : 往后(右)移动一个字符
  • Ctrl - xx : 在命令行尾和光标之间移动
  • Alt - b : 往回(左)移动一个单词
  • Alt - f : 往后(右)移动一个单词
  • M - b : 往回(左)移动一个单词
  • M - f : 往后(右)移动一个单词

编辑命令

  • Ctrl - h : 删除光标左方位置的字符
  • Ctrl - d : 删除光标右方位置的字符(注意：Ctrl - d 也代表EOF，当前命令行没有任何字符时，会注销系统或结束终端)
  • Ctrl - w : 由光标位置开始，往左删除单词(往行首删)
  • Alt - d : 由光标位置开始，往右删除单词(往行尾删)
  • M - d : 由光标位置开始，删除单词，直到该单词结束
  • Ctrl - u : 由光标所在位置开始，删除左方所有的字符，直到该行开始
  • Ctrl - k : 由光标所在位置开始，删除右方所有的字符，直到该行结束
  • Ctrl - y : 粘贴 Ctrl - u 或 Ctrl - k 剪切的内容到光标后
  • Ctrl - t : 交换光标处和之前两个字符的位置
  • Alt - . : 使用上一条命令的最后一个参数
  • Ctrl - _ : 回复之前的状态，撤销操作

搭建MCPE服务器

发表于 2018-04-23 | 分类于 电子游戏 | 评论数：
本文字数： 753 | 阅读时长 ≈ 1 分钟

目前搭建MCPE服务器的方法主要有两种：PocketMine-MP和Nukkit。前者基于PHP，而后者基于Java。
无论选择哪一种方法，都要确保游戏客户端和服务器端均是最新版本。如果版本不匹配，可能造成无法连接等问题。

PocketMine-MP

运行PocketMine-MP的方法非常简单：克隆这个项目，然后执行bash start.sh就可以了。但是，这样会有一个问题：PocketMine-MP需要php7环境和大量的拓展库，例如php-leveldb。通过brew或yum编译安装的php一般是不会自带这些拓展的，这样在启动时会报错。为了免去手动编译拓展的麻烦，解决方法是使用php-build-scripts。它会编译出带有所有必要的拓展库的php，供PocketMine-MP使用，并且不会与系统原有的php环境冲突。比如在Mac下，执行./compile.sh -t mac64 -l -j4 -c -f，完成后，再执行./installer.sh和./start.sh就可以运行了，根据提示进行配置即可。

Nukkit需要Java8环境，如果没有安装，可以前往ORACLE官网寻找帮助。运行方式也很简单，依次执行：

在其官网上也有发布的版本，无需自行编译Jar文件，可以前往Nukkit.io查看。
Nukkit的主要问题是不支持leveldb，也就是无法打开较新的MCPE存档，只支持MC存档和上古的MCPE存档。

客户端

PocketMine-MP和Nukkit启动后默认都会监听19132端口，配置服务器时填写局域网或公网IP/域名即可。

启动Apache

在终端输入：sudo apachectl start，即可启动Mac自带的Apache服务器。关闭和重启服务器的命令分别是sudo apachectl stop和sudo apachectl restart。
用浏览器打开http://localhost，若显示“It works!”，说明服务器启动成功。

配置服务器

Apache的配置文件为/etc/apache2/httpd.conf，为防止出现问题，请将其备份后进行编辑。如果更新了Mac系统，可能需要重新进行配置。
Apache默认的根目录在/Library/WebServer/Documents/下，你可以通过编辑DocumentRoot属性来修改它。

报错信息

如果使用sudo apachectl start命令启动Apache服务器，是不会显示错误信息的。因此，如果启动失败，还需要使用sudo /usr/sbin/httpd -k start查看报错信息。如果是httpd.conf配置出现问题，请对照报错的行数进行修改。如果是找不到目录等问题，检查相应的目录或权限即可。
